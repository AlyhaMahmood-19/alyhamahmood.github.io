    Working

    Create an empty folder anywhere on your PC and open it through VS Code. This is the folders in which we will keep all our files necessary to run our operating system

    First we have to create some assembly files, which will setup virtual addressing and long mode

    To combine all the object files into a single executable we need a linker.ld file, "linker.ld".

    Next step will be to create a main.c file which will print the image on to the screen.

    The final step before building the new operating system is to create a "Makefile".

    In order to test the OS, you need to have a virtual machine and that is Qemu.You can download Qemu from the top of the page.

  • Dockerfile

    A Dockerfile is a text document that contains all the commands a user could call on the command line to assemble an image. Using docker build users can create an automated build that executes several command-line instructions in succession.The code to be written in this file has been mentione above in Task 1.

  • header.asm

    This file contains the following code.

    section .multiboot_header

    header_start:

    ; magic number

    dd 0xe85250d6 ; multiboot2

    ; architecture

    dd 0 ; protected mode i386

    ; header length

    dd header_end - header_start

    ; checksum

    dd 0x100000000 - (0xe85250d6 + 0 + (header_end - header_start))

    ; end tag

    dw 0

    dw 0

    dd 8

    header_end:

  • main.asm

    This file contrains the assembly language code which is used to build the Operating System, it contains all the necesaary registers and commands for example, it has mov instructions, jmp instructions, loops and stack storage commands etc.

  • main.c

    This is the file which is used to un your code, to make a logo, add print statements to this file and run with the five commands again and you will get your desired image as output.
